--- layout: post title: JRuby bliss --- I've been fiddling with JRuby for about a month now and I have to say it's great. For those of you who don't know what it is - it's an implementation of the Ruby language written in Java. Most people are familiar only with the Matz's implementation in C (MRI = Matz's Ruby Interpreter also known as CRuby). Having Ruby available on top of the JVM is a big leverage. I was surprised how easy it was to incorporate Java code into Ruby code and vice versa. The most work I do is web development with Rails. JRuby supports Rails since version 0.9. Right now it runs all Rails test cases, is compatible partialy compatible with Ruby 1.9 and improved performance. The existing Rails applications I've worked on run significantly faster on JRuby than on CRuby (JVM rocks). You also get to compile your Ruby classes to Java classes or use your favourite Java libraries in you Ruby apps. The only downside at the moment is the Gem compatibility issues (not all Ruby code runs on JRuby, however can be adjusted in most cases). JRuby doesn't support the use of native extensions which in consequence forces you to swap some gems for others or to find pure Ruby (which means JRuby compatible) implementations. The bottom line is, JRuby provides you with exciting new possibilities. I'm looking forward to see the next releases by the JRuby team.
